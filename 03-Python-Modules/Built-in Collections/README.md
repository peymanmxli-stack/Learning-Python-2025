# 🧠 Python Built-in Collections

## 📌 Overview

Python provides several **built-in collection data types** used to store,
organize, and manage multiple values efficiently.

These collections are core components of **professional Python programming**
and are used internally by the language, standard library, and nearly all
real-world applications.

This module introduces Python’s built-in collections using their **official
category classification**: Sequences, Sets, and Mappings.

## 📦 Built-in Collection Categories

Python groups its built-in collections into **three main categories**:


Built-in Collections
│
├── Sequences
│ ├── list
│ ├── tuple
│ ├── str
│ └── range
│
├── Sets
│ ├── set
│ └── frozenset
│
└── Mappings
└── dict


Each category exists to solve **specific data modeling problems**.

---

## 🔹 1. Sequences

Sequences store **ordered collections of items**.
Each element has a position (index).

### Included Types:
- `list` → Mutable, ordered collection
- `tuple` → Immutable, ordered collection
- `str` → Immutable sequence of characters
- `range` → Immutable sequence of numbers (efficient iteration)

📌 Used when:
- Order matters
- Data is accessed by position
- Iteration is required

---

## 🔹 2. Sets

Sets store **unordered collections of unique elements**.

### Included Types:
- `set` → Mutable, no duplicates
- `frozenset` → Immutable set

📌 Used when:
- Uniqueness matters
- Fast membership testing is needed
- Mathematical set operations are required

---

## 🔹 3. Mappings

Mappings store **key–value pairs**.

### Included Types:
- `dict` → Mutable mapping of unique keys to values

📌 Used when:
- Data needs labels
- Fast lookup by key is required
- Modeling real-world entities (users, configs, records)
